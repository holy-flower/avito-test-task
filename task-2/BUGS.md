## BUG-001 POST /api/1/item возвращает тело ответа, не соответствующее контракту из Postman-коллекции

**Краткое описание:**  
Ручка создания объявления возвращает только поле `status` со строкой, содержащей id, вместо объекта объявления.

**Шаги воспроизведения:**  
1. Отправить POST-запрос на `https://qa-internship.avito.com/api/1/item` с валидным телом.
2. Проверить тело ответа.

**Фактический результат:**  
Возвращается:
```json
{"status": "Сохранили объявление - <id>"}
```
**Ожидаемый результат:**
Согласно Postman-коллекции, должен возвращаться объект объявления с полями:
id, sellerId, name, price, statistics, createdAt.

**Серьёзность:**
Major

**Окружение:**
- Host: https://qa-internship.avito.com
- Python 3.12
- pytest
- requests


## BUG-002 POST /api/1/item не отвечает на невалидный запрос без обязательного поля name

**Краткое описание:**  
При отправке запроса на создание объявления без обязательного поля `name` сервис не возвращает ошибку валидации и не отвечает в разумное время.

**Шаги воспроизведения:**  
1. Отправить POST-запрос на `https://qa-internship.avito.com/api/1/item`
2. Передать тело:
```json
{
  "sellerID": 257346,
  "price": 1000,
  "statistics": {
    "likes": 1,
    "viewCount": 10,
    "contacts": 1
  }
}
```
3. Дождаться ответа.

**Фактический результат:**
Сервис не отвечает более 30 секунд, запрос завершается по timeout.

**Ожидаемый результат:**
Сервис должен вернуть ошибку валидации (400 Bad Request или 422 Unprocessable Entity) за разумное время.

**Серьёзность:**
Major

**Окружение:**
- Host: https://qa-internship.avito.com
- Python 3.12
- pytest
- requests
