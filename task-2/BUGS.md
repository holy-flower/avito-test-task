## BUG-001 POST /api/1/item возвращает тело ответа, не соответствующее контракту из Postman-коллекции

**Краткое описание:**  
Ручка создания объявления возвращает только поле `status` со строкой, содержащей id, вместо объекта объявления.

**Шаги воспроизведения:**  
1. Отправить POST-запрос на `https://qa-internship.avito.com/api/1/item` с валидным телом.
2. Проверить тело ответа.

**Фактический результат:**  
Возвращается:
```json
{"status": "Сохранили объявление - <id>"}
```
**Ожидаемый результат:**
Согласно Postman-коллекции, должен возвращаться объект объявления с полями:
id, sellerId, name, price, statistics, createdAt.

**Серьёзность:**
Major

**Окружение:**
- Host: https://qa-internship.avito.com
- Python 3.12
- pytest
- requests


## BUG-002 Поле createdAt возвращается в некорректном формате

**Краткое описание:**  
В ответе API поле `createdAt` содержит некорректный формат даты и времени: timezone дублируется.

**Шаги воспроизведения:**  
1. Создать объявление через `POST /api/1/item`.
2. Получить объявление через `GET /api/1/item/{id}`.
3. Проверить значение поля `createdAt`.

**Фактический результат:**  
Поле `createdAt` возвращается в виде:
`2026-04-03 19:22:03.990706 +0300 +0300`

**Ожидаемый результат:**  
Поле `createdAt` должно возвращаться в корректном формате даты/времени с одним timezone, например:
`2026-04-03T19:22:03.990706+03:00`
или другом валидном и единообразном формате.

**Серьёзность:**  
Minor / Major  
(если контракт ответа важен — можно ставить Major)

**Окружение:**  
- Host: https://qa-internship.avito.com  
- Python 3.12  
- pytest  
- requests
