# Avito QA Internship Test Task

Автоматизированные тесты для API сервиса объявлений.

**Описание**

Проект содержит набор автотестов для проверки API сервиса объявлений.

Тесты покрывают основные сценарии работы API:

- создание объявления;
- получение объявления по id;
- получение объявлений по sellerID;
- получение статистики объявления;
- негативные сценарии (валидация входных данных);
- граничные значения;
- проверку структуры ответа API;
- базовые нефункциональные проверки.

Тесты написаны с использованием pytest и requests.

Используемые технологии
Python 3.12
pytest
requests
Структура проекта

**Test Strategy**

Тестирование API выполнялось по следующим направлениям.

**Функциональные проверки и позитивное сценарии**

Проверка основных бизнес-сценариев работы API:

- создание объявления;
- получение объявления по id;
- получение объявлений продавца;
- получение статистики объявления.

**Негативные сценарии**

Проверка обработки некорректных данных:

- отсутствие обязательных полей;
- неверные типы данных;
- некорректный формат id;
- запросы несуществующих ресурсов.

**Граничные значения**

Проверка граничных значений:

- минимальные допустимые значения;
- большие значения полей;
- граничные значения sellerID.

**Контрактное тестирование**

Проверка структуры ответа API:

- обязательные поля ответа;
- типы данных;
- структура объекта statistics.

**Нефункциональные проверки**

Дополнительные проверки:

- формат поля createdAt;
- заголовок Content-Type;
- время ответа API.

**Пример API запроса**

Создание объявления

POST /api/1/item

{
  "sellerID": 123456,
  "name": "Test item",
  "price": 1000,
  "statistics": {
    "likes": 1,
    "viewCount": 10,
    "contacts": 1
  }
}

Пример ответа

{
  "status": "Сохранили объявление - 123e4567-e89b-12d3-a456-426614174000"
}

После создания объявления выполняется запрос:

GET /api/1/item/{id}

для получения полной информации об объявлении.

1. Установка зависимостей

2. Запуск тестов
pytest -v

**Особенности реализации**

В проекте реализованы:

- использование requests.Session для переиспользования соединений;
- фикстуры pytest для подготовки тестовых данных;
- проверка контрактов API;
- нормализация ответа API;
- безопасный парсинг JSON (safe_json);
= проверка формата даты createdAt;

**Примечания**

API иногда возвращает данные в нестандартном формате.
Автотесты адаптированы под фактическое поведение сервиса.
